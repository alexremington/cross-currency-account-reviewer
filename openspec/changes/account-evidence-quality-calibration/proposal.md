# Account Evidence Quality And Renaissance Calibration

## Goal

Keep Account Website and Phone evidence comparable between Cross Currency Account Reviewer and Duplicate Reviewer while preserving raw invalid values for auditability, and allow the Renaissance Schools exact-name/address/phone bundle to promote despite a valid unequal Website conflict.

## Scope

- Add Account-only typed validation for Website and Phone with `valid`, `blank`, and `invalid` availability.
- Exclude invalid typed values from scoring, contradictions, corroboration, field statistics, blocking, and confidence rules while retaining raw values and invalid reasons.
- Preserve valid unequal Website and Phone values as conflicts by default.
- Narrowly permit the existing `exact-name-address-phone` rule when the only decisive conflict is a valid unequal Website and stronger same-level evidence is exact.
- Keep Contact behavior, free-text address validation, scheduler/producer behavior, and published artifact workflows unchanged.

## Calibration source

The local adjudication sample remains the source of human guidance. Representative adjudications will be added as protected fixtures after labels are returned.
